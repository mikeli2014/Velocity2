### RAG 设计(T4 重点)

详见之前 brainstorm(`08_待办路线图.md` § P1-1),确认细节:

1. **embedding 服务**:Voyage AI(`voyage-3-lite`,$0.02 / 1M tokens)。Anthropic 推荐,中文支持好,与现有 SDK 在同一生态。
2. **向量库**:numpy in-process(JSON 列存 embedding),~5K chunks 性能足够。**不引入 pgvector**(过度工程)。
3. **chunking**:600 字符 + 100 字符重叠,简单粗暴。
4. **检索**:`POST /api/v1/knowledge/search`,top-K=5。
5. **chat 集成**:新增 `retrieve: true` 标志,命中时把 chunks 作为 volatile system block 注入(在缓存 prefix 之后,避免破坏命中)。
6. **前端**:`AssistantChat` 加「📚 检索知识库」开关。
7. **补内容**:每条 KnowledgeSource 加「生成全文」按钮,Claude 一次性写入 ~1500 token Beihai 风的内容,然后 chunk → embed → 入库。

## 五、风险与注意事项

### 5.1 安全债

- **两把泄露的 Anthropic key 必须撤销**。即使现在没钱,key 仍是 PII。
- 长期方案:在 git pre-commit hook 加 `detect-secrets` 之类的扫描,避免再次发生。

### 5.2 部署习惯

- **永远用 Secret Manager 挂密钥**,不要再用 `--set-env-vars=KEY=value`,因为 `gcloud run services describe` 会把它原文打出来。
- 部署前 IAM 检查:`gcloud secrets get-iam-policy SECRET_NAME` 确认 service account 有 accessor 角色。

### 5.3 测试维护

- e2e 测试遇到的 selector 歧义都来自「同一个中文短语在多个 DOM 节点重复出现」。**新功能落地时**,测试用 `.tab`、`{exact:true}`、正则带后缀等手段提前规避。
- API integration 测试默认 skip,需要 `BASE_URL=https://...` 才跑。本地 dev 跑只看 23 个 features + smoke,不会误触发 prod 调用。

### 5.4 PowerShell 陷阱

- Windows 用 `Invoke-RestMethod` 不要用 `curl --data-raw '{...}'`,后者的引号会被 PS 吞掉。
- 写密钥用 `Out-File` → `--data-file=`,不要 `echo` 进 stdin(history 会留)。
