# 06 · 用户指南

本文档面向终端用户(高管、部门负责人、一线员工、知识官、IT 运营),按页面顺序介绍如何使用 Velocity OS。建议第一次打开系统时,点击首页的「🎬 5 分钟产品导览」走一遍内置导览。

## 0. 全局元素

### 顶部导航栏

- 左侧 **面包屑**:显示当前页面位置。
- 中部 **⌘K 搜索**(键盘快捷键 `Ctrl+K` 或 `⌘+K`):全局搜索 OKR / 项目 / 知识 / 部门 / 决策。回车跳转。
- 右侧:**API 状态徽章**(绿色 `API · sqlite`/`postgres` = 后端连接正常)、**通知铃铛**(红点 = 未读)、**新建按钮**。

### 侧栏导航

10 个顶层入口,部门工作空间下面是三级组织树。点击侧栏空白处或拖动滚动条可以导航。

### 移动端

宽度小于 768px 时,侧栏变为左侧抽屉。点击顶栏左侧的"☰ 汉堡按钮"打开,点击空白蒙层或任意菜单项关闭。多列网格自动折叠为单列。

## 1. 首页 · 企业认知总览

进入 `首页` 看到:

- **4 个顶部 KPI**:战略对齐率 / 知识复用率 / 项目健康度 / 当周 AI 调用。
- **关键 OKR 卡片**:每个 Objective 显示进度 + 关键风险灯。
- **战略议题预告**:in-debate 状态的 1 个战略问题。
- **活动流**:最近 8 条事件(实时审计 + 种子样例混合)。点击事件跳转对应资源。
- **部门状态**:每个一级部门的助手 / 知识 / 项目数。

> 💡 第一次访问时点击「🎬 5 分钟产品导览」按钮可以快速体验所有核心功能。已看过的用户按钮变为「再看导览」。

## 2. 公司知识中心

四个标签:

### 2.1 知识源 (Sources)

- 全部 ~8 条公司 / 部门级知识源,带类型徽章 (PDF / URL / DB)、范围、质量等级。
- 点击任意一行打开**详情模态**:摘要 + 节选 + 标签 + 反向引用列表(关联项目 / 决策)。
- 顶部 `已连接 API` 绿色 pill 表示数据来自 `/api/v1/knowledge-sources`(失败时回落种子)。
- 顶部 `+ 新增知识源` 按钮触发上传向导。

### 2.2 知识域 (Domains)

- 8 个域的健康灯 / 覆盖率仪表板。
- 点击域名进入域详情(数量、最近更新、标签云)。

### 2.3 知识图谱 (Graph)

- 当前为占位 SVG;未来 Phase 3 RAG 上线后会变为真实知识图谱。

### 2.4 上传队列 (Ingest Queue)

- 流水线状态:`fetching → parsing → embedding → tagging → review`。
- `review` 状态下,**审核员**可以点击「✓ 入库」完成入库(后端走 `/ingest-queue/{id}/approve` 端点),或「✗ 拒绝」。
- 失败行可点击「↻ 重试」。

## 3. 战略工作台

入门:第一次进入时建议先去「战略问题」标签选一个已有问题(如`FY26 是否加大线上 DTC 渠道投入?`)。或者点「+ 提出新问题」自己创建。

### 3.1 战略问题 (Registry)

- 5 个种子问题,带状态徽章(草稿 / 研讨中 / 已定调)。
- 点击行选中,自动跳到「画布」标签。

### 3.2 画布 (Canvas)

- 空间布局:中心是问题,内圈是 OKR,外圈是 Agent,左侧是上下文知识源。
- 工具栏可以「添加 Agent」「添加上下文」「重置画布」。

### 3.3 War Council (核心)

最重要的标签。

1. 顶部状态栏:`已运行 X 轮 · 共 N 条发言`,右侧两个按钮:「重新生成摘要」「运行第 X 轮」。
2. 主区域:按轮次分组展示历史发言,每条带 Agent 头像、立场徽章(绿赞成 / 红反对 / 黄保留)、引用知识源 chip。
3. 底部:**研讨摘要**面板,显示立场计数 + Sonnet 综合的 3-4 句中文结论。
4. 点击「运行第 X 轮」启动新一轮**流式辩论**:每个 Agent 头像在响应过程中亮起,正文逐 token 流入,完成后凝固。
5. 点击「重新生成摘要」运行综合调用(SSE),立场计数立即更新,文本流式刷新。

> 💡 一个问题的 Agent 列表为空(如 `sq-5`)时,默认使用全部 8 个 Agent,头部会有黄色提示「使用全部 8 个默认 Agent」。

### 3.4 战略选项

- 空状态:`本问题尚未生成战略选项`。点击右上角「生成战略选项」运行 Sonnet。
- 生成后:2-3 个候选方案卡片,带 ROI / 风险 / 节奏 / 赞成 / 反对 / ⭐ 推荐徽章。
- 点击「重新生成」覆盖现有选项。

### 3.5 结构化输出

- 空状态:`本问题尚未生成结构化输出`。点击「生成结构化输出」运行 Sonnet。
- 生成后:三个卡片 + 三个写入按钮:
  - **Objective + KRs** → 「写入 OKR 注册表」(POST `/objectives`),完成后变成 `✓ 已写入 OKR` + 「打开 OKR 页面 →」快捷键。
  - **关键项目** → 「写入 N 个项目」(逐个 POST `/projects`)。
  - **决策日志** → 「写入决策日志」(POST `/decisions`)。
- 已写入的项不再可重复点击。

## 4. OKR 与关键项目

四个标签:

### 4.1 公司 Objectives

- 4 个顶层 Objective,每个展开看到 KRs(`O1` 用户为中心、`O2` 渠道协同、`O3` CMF 中台、`O4` AI Native 等)。
- 每条 KR 旁有 **Check-in 按钮**(`↻` 图标),点击打开 KR 进度更新对话框,带历史曲线。
- 顶部「+ 新增 Objective」打开编辑器,可添加多个 KR;保存时调用 `POST /objectives`。

### 4.2 关键项目组合

- 7 个种子项目,按部门 / OKR / 健康灯过滤。
- 点击行打开**项目详情**:负责人、里程碑、风险列表、关联决策、关联知识源。
- 编辑器支持多里程碑 / 多风险条目;保存调用 `POST/PATCH /projects`。

### 4.3 战略对齐图

- Sankey 风格的 Objective × Project 关联图,鼠标悬停高亮链路。

### 4.4 决策日志

- 3 条种子决策。点击行打开详情:问题、结论、关键假设、反对意见、回顾。
- 「+ 记录决策」可手动新增;若来自战略工作台「写入决策日志」按钮,会自动填充关联 question + 假设 / 反对。

## 5. 部门工作空间

### 5.1 部门列表

- 三级树:中心 → 部门 → 团队。颜色按部门主色。
- 点击部门卡片进入详情。

### 5.2 部门详情(以工业设计部为例)

6 个标签:

- **概览**:KPI(项目 / 知识量 / 助手月活)+ 当前 OKR 关联。
- **知识中心**:本部门可见的知识源子集。
- **CMF Intelligence** *(仅工业设计部)*:CMF 标准库 + 趋势看板。
- **Market Insights** *(仅工业设计部)*:奥维数据快照。
- **技能**:本部门 Skill Pack 列表,点击「启动 →」打开 RunDialog。
- **工作流**:本部门工作流模板。
- **助手**:与该部门助手(`小龙虾`等)的 1:1 聊天界面。**输入框敲回车发送,响应实时流式**。
- **项目**:本部门承担的项目(可在此 CRUD,与 OKR 页同步)。

### 5.3 RunDialog(技能 / 工作流运行)

- 弹出大模态,顶部输入框 + 步骤进度条。
- 启动后 Sonnet 流式输出结果到底部「输出」面板。
- 完成后底部按钮:「复制」「保存为知识条目」「写回项目」「发送给助手」。

## 6. 助手中心

三大块:

### 6.1 助手卡片

- **企业战略助手 Velocity**(深紫):CEO/VP 入口,注入全公司 OKR + 项目。
- **5 个部门助手**:点击卡片直接跳转到对应部门工作空间的助手 Tab。

### 6.2 路由测试 · Haiku 4.5 实时分类

- 输入框输入用户提问(如 `PVD 工艺与喷涂的成本对比?`)。
- 4 个示例 chip 一键填充。
- 点击「测试路由」,Haiku 返回:
  - 命中规则编号 + 意图 + 目标部门 + 默认 Skill。
  - 置信度百分比。
  - 中文分类理由。
  - 调用模型显示在底部。

### 6.3 意图路由规则

- 8 条种子规则,按优先级排序。
- 列表支持启用 / 停用切换、上下移动、新增 / 编辑 / 删除。
- 每次写操作都通过 `POST/PATCH/DELETE /routing-rules` 同步,审计日志记录 `category=routing`。

## 7. 权限与治理

- 上半部分:角色矩阵 + 数据连接器(企业微信 / 钉钉 / 飞书示例)。
- 下半部分:**审计日志**——分类 / 严重度 / 关键词三层过滤,导出 CSV,点击行跳转到资源。

## 8. 管理后台

5 个标签:

### 8.1 Token 用量

- **顶部:实时 AI 用量面板**(本次重点新增)
  - 4 个 KPI:累计调用 / 输入 token / 输出 token / 缓存命中率(随比例变绿 / 黄 / 灰)。
  - 按模型 / 按路由两个汇总表。
  - 最近 10 次调用明细(状态、延迟、token、缓存读)。
  - 刷新按钮重新拉取 `/api/v1/llm-usage/summary`。
- **下半部分:演示历史数据**(seed)
  - 时间范围切换(今日 / 本周 / 本月 / 季度 / 年度)。
  - 按部门 / 按用户 Top 排行。

### 8.2 模型与路由

- 6 个 LLM 候选(Sonnet 4.6、Haiku 4.5、本地模型等)。
- 策略路由表:不同部门 / 场景对应不同模型。

### 8.3 组织结构

- 三级组织树编辑器,点击节点查看详情。

### 8.4 配额与预算

- 月度预算 / 当前消耗 / 月底预测。
- 部门配额表,带告警阈值。

### 8.5 审计与日志

- 与「权限与治理 → 审计日志」类似,但偏运维视角。

## 10. 常见问题

**Q: 某个 AI 功能返回"未配置 ANTHROPIC_API_KEY"怎么办?**
A: 联系 IT 在 Cloud Run 服务上挂接 secret(参见 `cloudbuild.yaml` 注释),或在本地开发环境 `export ANTHROPIC_API_KEY=...`。

**Q: 创建的项目 / 决策刷新后消失?**
A: Cloud Run 实例冷启动后 SQLite 重置。如果需要持久化,要求 IT 切换到 Cloud SQL Postgres(参见 `03_技术架构.md` §6.3)。

**Q: 缓存命中率为什么这么低?**
A: 看管理后台「Token 用量 → 实时面板」。如果按模型 / 路由分布看不到 cache_read 数字,说明:
- 调用次数太少(前 1-2 次只创建缓存);
- 或者 prompt 中混入了易变内容(时间戳 / per-request ID)。
联系工程团队检查 `chat.py` / `debate.py`。

**Q: 文档上传失败怎么办?**
A: 上传队列页点失败行的「↻ 重试」;若反复失败,看错误信息,常见原因:文件过大(> 50MB)、格式不支持、字符编码异常。
