# 02 · 产品需求文档 (PRD)

本文档描述 Velocity OS 已交付的功能集合、用户角色、核心场景与功能边界。是产品 / 业务方对系统能力的完整说明。

## 1. 用户角色

| 角色 | 典型场景 | 关键页面 |
|---|---|---|
| **CEO / 高管** (陈志远) | 提出战略问题、看 OKR 执行状况、读决策日志、查询全公司知识 | 首页 / 战略工作台 / OKR / 助手中心 |
| **VP / 部门负责人** (李慕白等) | 维护部门 OKR、主持 War Council、配置部门助手 | OKR / 战略 / 部门工作空间 |
| **一线员工** (设计师 / 服务工程师) | 用部门助手解决具体问题、调用 Skill Pack、上传知识 | 部门工作空间 / 工作流中心 |
| **知识官** (苏婉等) | 审核入队文档、维护知识域、监控质量等级 | 公司知识中心 |
| **IT / AI 运营** (张毅) | 配置路由规则、查看用量、配额、审计 | 管理后台 / 助手中心 |

## 2. 信息架构

10 个顶层页面,对应代码 `src/pages/`:

```
首页                  HomePage         企业认知总览(KPI、活动、研讨预告)
公司知识中心          KnowledgePage    8 个知识域 + 知识源列表 + 上传队列 + 知识图谱
战略工作台            StrategyPage     问题注册表 + 画布 + War Council + 选项 + 结构化输出
OKR 与关键项目        OkrPage          Objectives + Projects + 决策日志 + 战略对齐图
部门工作空间(列表)   DepartmentsIndex 三级组织树概览
部门工作空间(详情)   DepartmentPage   单部门:概览 / 知识 / 技能 / 工作流 / 助手聊天 / 项目
技能中心              SkillsPage       全部 Skill Packs 列表 + CRUD + 运行
工作流中心            WorkflowsPage    工作流模板 + 运行历史 + 技能×知识域映射
助手中心              AssistantsPage   助手卡片 + 路由规则 CRUD + 路由测试
权限与治理            GovernancePage   RBAC + 数据连接器 + 审计日志
管理后台              AdminPage        Token 用量 + 模型路由 + 组织 + 配额 + 审计
```

## 3. 核心场景

### 场景 A · CEO 启动一次战略研讨

**用户故事**:作为陈志远(CEO),我要决定 FY26 是否加大线上 DTC 渠道投入,需要听各部门视角并产出可落地方案。

**步骤**:

1. 进入「战略工作台 → 战略问题」,点击「+ 提出新问题」,填写问题、关联 OKR、选择 5-7 个 Agent 角色、选择 3 条背景知识。
2. 切到「画布」,确认上下文已注入(公司 OKR + 部门 + Agent 排布在空间画布上)。
3. 切到「War Council」,点击「运行第 1 轮」。每个 Agent 以流式 token 输出立场(`[赞成]`/`[反对]`/`[保留]`)+ 4 句话理由,引用知识源。
4. 第 1 轮完成后,可以点击「重新生成摘要」让 Sonnet 综合各方立场,显示「3 赞成 / 1 反对 / 3 保留」+ 一段核心张力分析。
5. 点击「运行第 2 轮」(交叉质询)、「运行第 3 轮」(立场收敛)。
6. 切到「战略选项」,点「生成战略选项」,Sonnet 综合辩论生成 2-3 个候选方案。
7. 切到「结构化输出」,点「生成结构化输出」,得到 Objective + 2-3 个 KR + 2-3 个项目 + 1 条决策日志草案。
8. 分别点击「写入 OKR 注册表」「写入 N 个项目」「写入决策日志」,数据落到 Objective / Project / Decision 表,可在 OKR 页面看到新行。

### 场景 B · 设计师向部门助手提问

**用户故事**:作为工业设计部 CMF 中台设计师,我要了解 PVD 工艺与喷涂在零冷水热水器外壳上 500 台试产成本对比。

**步骤**:

1. 进入「部门工作空间 → 工业设计部」,Tab 切到「助手」。
2. 输入框输入问题,回车。
3. 助手「小龙虾」(Sonnet 4.6)以流式 token 实时回复,引用知识库(ks-5 / ks-8)。
4. 回复气泡下方有「关联 O1, O3」「✦ 供应商/材料/工艺检索」标签,可点知识源 chip 跳转知识中心查看原文。
5. 点回答下方的「+ 补充知识」可把问答保存为新知识条目,「↗ 写入项目」可关联到具体项目。

### 场景 C · 知识官审核新上传文档

**用户故事**:作为 CMF 中台主管,我要把一份新到的供应商报价 PDF 入库为公司知识。

**步骤**:

1. 进入「公司知识中心」,点击「+ 新增知识源」选 PDF / URL / 数据库连接,文件进入 Ingest Queue。
2. 系统流水线显示状态:`fetching → parsing → embedding → tagging → review`。
3. 切到「上传队列」标签,看到 review 状态的文件,点「入库」按钮。
4. `POST /api/v1/ingest-queue/{id}/approve` 把队列项变成正式 KnowledgeSource 行,审计日志记录「知识源入库」。
5. 该文件即出现在「知识源」标签页,可被任意助手 / 技能引用。

### 场景 D · IT 配置路由规则

**用户故事**:作为 IT 平台,我要让"PVD"开头的提问路由到工业设计部小龙虾助手。

**步骤**:

1. 进入「助手中心」,看到「路由测试 · Haiku 4.5 实时分类」面板。
2. 点「+ 新增规则」,填写意图模式("CMF / 材料 / 工艺 / 色彩 / 表面")、目标部门(工业设计部)、默认 Skill(供应商/材料/工艺检索)、权限(vp,lead,staff)。
3. 保存后立即在测试面板输入"PVD 工艺与喷涂的成本对比?",看到 Haiku 4.5 返回 `rule_id=rt-1, confidence=0.92, rationale=匹配 CMF 工艺关键词`。
4. 审计日志同时记录「新增路由规则」。

## 4. 功能清单(已交付)

### 4.1 增删改查 (CRUD)

| 实体 | 端点 | 前端入口 | 审计 |
|---|---|---|---|
| Objective + KR | `GET/POST/PATCH/DELETE /objectives` | OkrPage | (内置 KR diff) |
| Project | `GET/POST/PATCH/DELETE /projects` | OkrPage | category=project |
| Decision | `GET/POST/PATCH/DELETE /decisions` | OkrPage | category=decision |
| KnowledgeDomain | `GET /knowledge-domains` + `PATCH /knowledge-domains/{id}` | KnowledgePage | — |
| KnowledgeSource | `GET/POST/PATCH/DELETE /knowledge-sources` | KnowledgePage(只读 + 通过 ingest 入库) | category=knowledge |
| RoutingRule | `GET/POST/PATCH/DELETE /routing-rules` | AssistantsPage | category=routing |
| IngestQueue | `GET/POST/PATCH /ingest-queue` + `POST /ingest-queue/{id}/approve` | KnowledgePage | (approve 时审计) |
| Notification | `GET /notifications` + `POST /notifications/{id}/read` + `POST /notifications/mark-all-read` | Topbar | — |
| StrategyOption | `GET/POST /strategy-questions/{id}/options` | StrategyPage | — |
| DebateMessage | `GET /strategy-questions/{id}/debate` | StrategyPage(自动写入) | — |

只读列表:`/company`、`/departments`、`/skill-packs`、`/workflows`、`/workflow-runs`、`/activity`、`/agents`、`/strategy-questions`、`/audit-log`。

### 4.2 AI 端点

| 端点 | 模型 | 流式 | 用途 |
|---|---|---|---|
| `POST /chat` | Sonnet 4.6 | 否 | 部门助手单轮回复 |
| `POST /chat/stream` | Sonnet 4.6 | SSE | 部门助手流式回复 |
| `POST /strategy-questions/{id}/debate/round` | Sonnet 4.6 | 否 | 多 Agent 辩论一轮(同步) |
| `POST /strategy-questions/{id}/debate/round/stream` | Sonnet 4.6 | SSE | 多 Agent 辩论一轮(流式,逐 Agent) |
| `POST /strategy-questions/{id}/debate/synthesis` | Sonnet 4.6 | 否 | 辩论综合摘要 |
| `POST /strategy-questions/{id}/debate/synthesis/stream` | Sonnet 4.6 | SSE | 辩论综合摘要(流式) |
| `POST /strategy-questions/{id}/options/generate` | Sonnet 4.6 | 否 | 生成 2-3 个战略选项(JSON) |
| `POST /strategy-questions/{id}/structured-output/generate` | Sonnet 4.6 | 否 | 生成 Objective + KR + 项目 + 决策草案(JSON) |
| `POST /route` | Haiku 4.5 | 否 | 意图分类到路由规则 |

所有 AI 端点都使用 Anthropic prompt caching;未配置 `ANTHROPIC_API_KEY` 时返回 `503 anthropic_not_configured`,前端软退化为种子内容。

### 4.3 治理与审计

- **X-User-Id 头**:每个写请求可携带 ASCII 用户标识(默认 `陈志远`),自动写入审计日志的 `actor` 字段。
- **审计分类**:`project` / `decision` / `knowledge` / `routing` / `okr` / `assistant` / `skill` / `config`。
- **CSV 导出**:权限与治理页 + 审计日志支持过滤后导出 CSV。
- **Token 用量面板**:管理后台「Token 用量」标签实时展示按模型 / 路由的调用统计 + 缓存命中率 + 最近 20 次调用。

## 5. 已知限制 / 暂不支持

| 限制 | 影响 | 何时解决 |
|---|---|---|
| 没有 RAG | 助手回复基于公司 / 部门概要而非全文检索 | Phase 3,需先决定 embedding 服务 + 向量库 |
| Cloud Run 默认 SQLite | 冷启动后用户产生的数据丢失 | 切到 Cloud SQL Postgres,代码已就绪 |
| 没有真实 OAuth | `X-User-Id` 是开放头,任意值都能写入审计 | Phase 3,通过同一个 `current_user` dep 接入 OAuth |
| 反对意见 / 假设是文本 | 没有结构化追踪反对者后续的态度变化 | 暂不在路线图 |
| 没有移动端原生 App | 仅响应式 Web | 路线图待评估 |

## 6. 验收标准

下面这些行为必须工作(已通过 127 个 pytest + 29 个 Playwright 测试):

1. 点击侧栏「公司知识中心」能看到「已连接 API」绿色 pill。
2. 在「战略工作台」对一个空白战略问题点「运行第 1 轮」,看到所有 Agent 的头像在响应过程中亮起。
3. 「结构化输出」点击「写入 OKR 注册表」后,跳到 OKR 页面能看到新行。
4. 「助手中心」的路由测试输入"PVD 成本",至少返回一个匹配规则 + 置信度。
5. 「管理后台 → Token 用量」第一行(实时面板)显示真实数字,缓存命中率随调用次数变化。
