## 1. 整体架构

```

│                      Cloud Run · velocity2                       │
│                                                                  │

│   │   /api/v1/*             │     └─────────────────────────┘    │
│   │   /healthz              │                                    │

│            │                                                     │
│            ▼                                                     │
│   ┌─────────────────────────┐     ┌─────────────────────────┐    │
│   │  SQLite (默认 /         │     │  Anthropic API          │    │

│   └─────────────────────────┘     └─────────────────────────┘    │

                       ▲
                       │ HTTPS
                       │
                ┌──────┴───────┐
                │  浏览器      │
                │  (Web/移动)  │
                └──────────────┘
```

**核心决策**:**单一 Cloud Run 服务**承载前后端,镜像两阶段构建——Stage 1 用 `node:20-alpine` 跑 `npm run build`,Stage 2 用 `python:3.11-slim` 安装 FastAPI 依赖并 `COPY --from=frontend /web/dist ./static`。运行时 FastAPI 提供 `/api/v1/*`,catch-all 把其他路径回落到 `static/index.html`(SPA 路由)。

## 2. 前端

| 项 | 选择 | 原因 |
|---|---|---|
| 构建 | Vite 8 | 快速冷启动,esbuild 转译 |
| 框架 | React 19 + 新 JSX 转译 | `import React` 不强制使用 |
| 路由 | 自研 `setRoute` 状态机 | SPA,不需要 URL 深链;Cmd-K 面板覆盖搜索 |
| 状态 | 原生 React state + `window.dispatchEvent` 自定义事件 | 项目规模不需要 Redux/Zustand |
| CSS | 单一手写 `src/styles/index.css`(token + 工具类) | 不引入 Tailwind/Bootstrap |
| 图标 | 自研 SVG icon map (`primitives.jsx`) | 体积可控,样式统一 |
| HTTP 客户端 | 自研 `src/lib/api.js` (`useApi`/`apiPost`/`apiStream`) | 同源 + Vite proxy → :8081 |
| 代码分割 | `React.lazy` (Strategy / Workflows / Admin) | 首屏 < 110 KB gzip |

## 5. 关键设计决策

### 5.1 Prompt Caching(头等关注点)

每个 AI 调用的 system 字段都是 `list[TextBlockParam]`,而不是单字符串:

```python
system_blocks = [
    {"type": "text", "text": company_block_text, "cache_control": {"type": "ephemeral"}},  # 公司事实(稳定)
    {"type": "text", "text": dept_block_text,    "cache_control": {"type": "ephemeral"}},  # 部门人设(稳定)
    {"type": "text", "text": skill_block_text,   "cache_control": {"type": "ephemeral"}},  # 技能框架(稳定)
    {"type": "text", "text": volatile_suffix},                                             # 易变(不缓存)
]
```

辩论场景中,7 个 Agent 调用共享 `[Company → Question]` 前缀,只在最后的 Agent 人设处分叉,因此缓存命中率显著提升。`AdminPage` 的实时面板可以看到。

### 5.2 流式 SSE

`/chat/stream`、`/debate/round/stream`、`/debate/synthesis/stream` 都用 FastAPI `StreamingResponse` + `text/event-stream` 媒体类型,事件格式:

```
data: {"type":"text","text":"片段..."}\n\n
data: {"type":"agent_start","agentId":"ag-product"}\n\n
data: {"type":"agent_done","agentId":"ag-product","message":{...}}\n\n
data: {"type":"counts","pro":3,"con":1,"concern":3}\n\n
data: {"type":"done","model":"...","usage":{...}}\n\n
data: {"type":"error","detail":"anthropic_rate_limited"}\n\n
```

前端通过 `apiStream(path, body, {onEvent})`(在 `src/lib/api.js`)消费,React 组件根据 `type` 分派到 UI 状态。

### 5.3 软退化(Graceful Degradation)

未配置 `ANTHROPIC_API_KEY` 时:

- AI 端点返回 `503 anthropic_not_configured`(稳定 slug,前端可本地化)。
- 前端检测此 slug,展示种子内容 + 黄色提示横幅,确保演示从不"白屏"。

类似的,API 完全离线时,所有 `useApi` 钩子降级为 `src/data/seed.js` 的种子数据。

### 5.4 审计日志中心化

`emit_audit(db, category, severity, action, target, scope, actor, link)` 是所有写端点共享的 helper,位于 `velocity_api/audit.py`。每次 CRUD / 治理动作都通过它落库,前端审计日志页可点击行跳转。

`actor` 字段从 `X-User-Id` HTTP 头取(经 `auth.current_user` 依赖),无头时回落 `DEFAULT_ACTOR = "陈志远"`。

### 5.5 LLM 遥测

`record_llm_call(db, route, model, latency_ms, usage, status, error_detail)` 在每次 Anthropic SDK 调用前后采样,写入 `LLMCall` 表。汇总端点 `GET /api/v1/llm-usage/summary` 计算缓存命中率:

```
hit_ratio = cache_read / (cache_read + cache_creation + uncached_input)
```

## 6. 部署

### 6.1 流水线

`cloudbuild.yaml` 步骤:

1. `gcr.io/cloud-builders/docker build` 构建统一镜像。
2. `docker push` 到 Container Registry。
3. `gcloud run deploy velocity2` 部署到 us-east1。
4. `mcr.microsoft.com/playwright:v1.59.1-jammy` 跑 `npx playwright test --project=chromium` 烟雾测试。
5. (可选)上传 Playwright HTML 报告到 GCS。

### 6.2 环境变量

| 变量 | 默认 | 用途 |
|---|---|---|
| `VELOCITY_API_DATABASE_URL` | `sqlite:///./.data/velocity.db` | 数据库 URL |
| `VELOCITY_API_AUTO_SEED` | `true` | 启动时自动加载种子数据 |
| `VELOCITY_API_ALLOWED_ORIGINS` | localhost:5173 + Cloud Run URL | CORS |
| `ANTHROPIC_API_KEY` | 未设置 | AI 端点功能开关 |
| `PORT` | 8080 | Cloud Run 注入 |

### 6.3 切换到 Postgres

```bash
gcloud sql instances create velocity-db --tier=db-f1-micro --region=us-east1
gcloud sql databases create velocity --instance=velocity-db
gcloud run services update velocity2 \
  --region=us-east1 \
  --add-cloudsql-instances=PROJECT:us-east1:velocity-db \
  --update-env-vars=VELOCITY_API_DATABASE_URL=postgresql+psycopg://USER:PASS@/velocity?host=/cloudsql/PROJECT:us-east1:velocity-db
```

代码侧无需改动:`db._make_engine` 已经识别 `postgresql://` 前缀;`psycopg[binary]` 已在 `requirements.txt`;`libpq5` 已装在镜像。
